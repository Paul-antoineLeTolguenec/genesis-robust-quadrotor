## 3.1 ObsFixedDelay

### Description

Delays the observation by a fixed number of steps, sampled once per episode.
Uses a circular `DelayBuffer` to store and retrieve past observations.
delay=0 is a pass-through (no latency added).

**Hook timing:** `on_observation` — applied after sensor pipeline.

---

### Formal definition

At episode $k$, for each environment $i$:

$$
d_i^{(k)} \sim \text{round}\left(\mathcal{U}(\text{low}, \text{high}) \cdot s\right), \quad d_i^{(k)} \in [0, \text{max\_delay}]
$$

At each step $t$:

$$
\text{obs}^{\text{delayed}}_{i,t} = \text{obs}_{i, t - d_i^{(k)}}
$$

Cold start: buffer is zero-initialized; reads before first fill return zeros.

---

### Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `obs_slice` | slice | — | Observation channels to delay |
| `obs_dim` | int | — | Width of the obs slice |
| `max_delay` | int | `10` | Maximum delay in steps |
| `distribution_params` | dict | `{"low": 0, "high": 10}` | Integer delay bounds |
| `bounds` | tuple | `(0.0, 10.0)` | Hard clamp on delay |
| `nominal` | float | `0.0` | No-delay baseline |

## 3.2 ObsVariableDelay

### Description

Same mechanism as ObsFixedDelay but the integer delay is resampled every step,
producing time-varying observation latency.

---

### Formal definition

At each step $t$, for each environment $i$:

$$
d_{i,t} \sim \text{round}\left(\mathcal{U}(\text{low}, \text{high}) \cdot s\right), \quad d_{i,t} \in [0, \text{max\_delay}]
$$

$$
\text{obs}^{\text{delayed}}_{i,t} = \text{obs}_{i, t - d_{i,t}}
$$

---

## 3.3 ActionFixedDelay

### Description

Delays the action by a fixed number of steps, sampled once per episode.
Uses a circular `DelayBuffer`. delay=0 is a pass-through.
Zero-order hold on cold start (buffer zero-initialized).

**Hook timing:** `on_action` — applied before motor command.

---

### Formal definition

$$
d_i^{(k)} \sim \text{round}\left(\mathcal{U}(\text{low}, \text{high}) \cdot s\right)
$$

$$
\text{action}^{\text{delayed}}_{i,t} = \text{action}_{i, t - d_i^{(k)}}
$$

---

## 3.7 PacketLoss

### Description

Simulates network packet loss on the control command channel.
Each step, independently per environment, the action is dropped with
probability $p$ (sampled per step). When dropped, the previous valid
action is held (zero-order hold).

**Hook timing:** `on_action`.

---

### Formal definition

$$
p_i \sim \mathcal{U}(\text{low}, \text{high}) \cdot s, \quad p_i \in [0, 0.3]
$$

$$
\text{drop}_{i,t} \sim \text{Bernoulli}(p_i)
$$

$$
\text{action}^{\text{out}}_{i,t} = \begin{cases}
\text{action}^{\text{last\_valid}}_i & \text{if } \text{drop}_{i,t} = 1 \\
\text{action}_{i,t} & \text{otherwise}
\end{cases}
$$

---

### Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `action_dim` | int | `4` | Action vector width |
| `distribution_params` | dict | `{"low": 0.0, "high": 0.3}` | Drop probability range |
| `bounds` | tuple | `(0.0, 0.3)` | Hard clamp on drop probability |
| `nominal` | float | `0.0` | No-drop baseline |

### Catalog reference

Section 3.7 of `docs/01_perturbations_catalog.md`.

## 3.8 ComputationOverload

### Description

Simulates computation overload events where the on-board controller stalls
for a random number of steps. During a stall, the last valid action is
held (zero-order hold).

Each step:
- Active counters are decremented first
- For idle envs (counter=0), a Bernoulli trigger determines if a new stall starts
- If triggered, a random duration is sampled

---

### Formal definition

$$
p_i \sim \mathcal{U}(\text{prob\_low}, \text{prob\_high}) \cdot s
$$

$$
\text{trigger}_{i,t} \sim \text{Bernoulli}(p_i) \cdot \mathbb{1}[\text{counter}_i = 0]
$$

$$
D_{i,t} \sim \mathcal{U}_\text{int}(\text{dur\_low}, \text{dur\_high}) \quad \text{if triggered}
$$

$$
\text{counter}_{i, t+1} = \begin{cases}
D_{i,t} & \text{if triggered} \\
\max(\text{counter}_i - 1, 0) & \text{otherwise}
\end{cases}
$$

---

### Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `action_dim` | int | `4` | Action vector width |
| `distribution_params` | dict | `{"prob_low": 0.0, "prob_high": 0.1, "duration_low": 1, "duration_high": 5}` | Event parameters |
| `bounds` | tuple | `(0.0, 0.1)` | Hard clamp on skip probability |

### Catalog reference

Section 3.8 of `docs/01_perturbations_catalog.md`.
